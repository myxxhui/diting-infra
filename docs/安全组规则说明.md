# 安全组规则说明（prod 生产数据环境）

部署 prod 环境后，本机或 CI 需通过 EIP + NodePort 访问 TimescaleDB、PostgreSQL L2、Redis，**安全组须放通对应入站端口**。deploy-engine 的 Terraform 已放通：**22（SSH）、6443（K8s API）**、VPC 内网、出站全部；**NodePort 需按本配置在部署后放通**。

## 配置与执行流程

| 步骤 | 说明 |
|------|------|
| 1. 配置来源 | **config/security-group-rules.yaml**：端口、协议、源 CIDR、说明；需新增端口时直接编辑该文件。 |
| 2. 部署时更新 | 执行 **`make deploy diting prod`** 时，在 Terraform 创建 ECS/安全组后会自动执行 **`make apply-security-group-rules`**，按 YAML 为当前安全组添加入站规则。 |
| 3. 单独更新 | 若仅需更新安全组（未重新 deploy），在 diting-infra 根目录执行：**`make apply-security-group-rules`**。 |

## 当前需放通的端口（与 prod.conn / Bitnami 一致）

| 端口 | 协议 | 用途 | 说明 |
|------|------|------|------|
| 22 | tcp | SSH | 已由 deploy-engine Terraform 放通（ssh_allowed_cidr 或 apply 时出口 IP） |
| 6443 | tcp | K8s API | 已由 deploy-engine Terraform 放通 |
| 30001 | tcp | TimescaleDB L1 | prod-write-conn 默认 NODEPORT_L1；本机 `make verify diting prod` 需连通 |
| 30002 | tcp | PostgreSQL L2 | prod-write-conn 默认 NODEPORT_L2 |
| 30379 | tcp | Redis | 本机验证与采集客户端需连通 |
| 30432 | tcp | TimescaleDB L1 | Bitnami Chart 固定 NodePort 时使用 |
| 30433 | tcp | PostgreSQL L2 | Bitnami Chart 固定 NodePort 时使用 |

**源 CIDR**：当前配置使用 `0.0.0.0/0` 便于本机/任意环境验证；生产可改为本机 IP/32 或 CI 出口 IP。

## 依赖与未安装时的处理

- **apply-security-group-rules** 依赖 **aliyun CLI** 与已配置的默认地域/账号；脚本会从 deploy-engine 的 Terraform output 读取 `security_group_id`，再调用 `aliyun ecs AuthorizeSecurityGroup` 添加规则。
- 若未安装 aliyun CLI 或执行失败，请根据 **config/security-group-rules.yaml**（或上表）在阿里云控制台 **ECS → 安全组** 中手动添加入站规则：端口范围 = 上表端口，授权对象 = 0.0.0.0/0（或指定 IP），协议 TCP。

## 与 06_ 实践文档的关系

本配置解决「**make verify diting prod 报错：连接 EIP:30001 超时**」类问题：部署后安全组未放通 NodePort 会导致本机无法连通 L1/L2/Redis。按本文档配置并执行 **make apply-security-group-rules**（或部署时自动执行）后，再执行 **make verify diting prod** 即可通过。
